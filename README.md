#📘 **README — Prompts Técnicos J.A.R.V.I.S**
  Este repositório contém prompts estruturados criados para apoiar desenvolvedores em três modos distintos de trabalho: PLAN, STUDY e CODE.
  Eles foram desenhados para funcionar como copilotos técnicos, ajudando a organizar ideias, aprender conceitos e planejar implementações de forma clara e incremental.

##🎯 ***Objetivo dos Prompts***
  Os prompts têm como missão reduzir fricção no processo de desenvolvimento.
  Eles ajudam a transformar tarefas complexas em planos revisáveis, explicações didáticas ou implementações guiadas, sempre com foco em boas práticas e stacks modernas como Java 21 + Spring Boot 3.x.

##🧩 **Modos de Uso**

  ###1. Modo ***PLAN*** 📊
     
  -**Foco:** planejamento de implementação.
  
  -**Saída:** plano incremental com escopo, riscos, arquivos afetados e estratégia de testes.
  
  -**Benefício:** evita começar a codar sem clareza, reduz retrabalho e melhora a comunicação em equipe.

  ###2. Modo ***REFELCTIVE PRACTICE*** 🧙‍♂️
     
  -**Foco:** aprendizado e revisão de conceitos.
    
  -**Saída:** explicações progressivas (simples → avançado), analogias, exemplos mínimos e checkpoints de compreensão.
    
  -**Benefício:** acelera o aprendizado de devs em qualquer nível, trazendo intuição e prática sem excesso de teoria.
  
  ###3. Modo ***AGENT CODE*** (quando solicitado) 🤖
     
  -**Foco:** implementação didática.
    
  -**Saída:** código comentado, com explicação do porquê de cada decisão.
    
  -**Benefício:** gera exemplos práticos que podem ser adaptados diretamente em projetos reais.

##⚙️ **Stack Utilizada**
  Os prompts foram adaptados para refletir o ecossistema moderno de Java + Spring Boot, incluindo:

  -**Java 21**
  
  -**Spring Boot 3.x**
  
  -**PostgreSQL + Spring Data JPA**
  
  -**Spring Security + JWT**
  
  -**Springdoc OpenAPI (Swagger)**
  
  -**JUnit 5 + Mockito**
  
  -**Micrometer + Prometheus + Grafana**
  
  -**Kafka/RabbitMQ (quando necessário)**
  
  -**Redis para cache**
  
  -**Docker + Kubernetes para deploy**
  
  -**Spring Cloud Config para configuração distribuída**

##🚀 **Como Isso Ajuda no Dia a Dia**

  *Codar:* - *fornece planos claros antes de escrever código, evitando erros comuns.*

  *Aprender:* - *explica conceitos com progressão didática, analogias e exemplos práticos.*

  *Planejar:* - *organiza tarefas em passos pequenos e revisáveis, com riscos e validações mapeados.*

***Em resumo, os prompts funcionam como um copiloto técnico que guia o desenvolvedor em cada etapa: pensar, aprender e executar com segurança.***
